FaceIQ

A web app flow for checking whether a driver’s licence image, live face captures, and native Face ID / Android biometric presence line up.

Demo boundary: this is a public prototype surface. It does not store uploads, does not contact a server, and uses lightweight browser-side scoring only. Production identity, document-fraud, privacy, and biometric architecture would need hardened implementation before real-world reliance.

Start the camera to begin.

How the flow works

  1. The browser opens the FaceIQ challenge web app.
  2. The user photographs their driver’s licence.
  3. The app captures a live face image immediately before the biometric step.
  4. A native Face ID / Android biometric prompt should appear. Approve it on the device.
  5. The app captures another live face image immediately after that prompt completes or times out.
  6. A confidence level is displayed, including licence-image quality/fake-document signals.
